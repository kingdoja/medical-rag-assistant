# Demo Data Checklist

## 目标

定义医疗方向 demo 数据的准备、命名、去敏和入库前检查规范。

当前版本目标：为 `v0.1` 最小可演示版本准备一套公开、低风险、可稳定导入的数据包。

## 数据类型

建议至少准备以下 4 类：

1. 指南 / 规范
2. SOP / 制度
3. 设备说明书
4. 培训资料

## 允许来源

- 官方公开指南
- 医学会 / 行业协会公开资料
- 公开设备说明书
- 公开培训材料
- 去敏后的教学资料

## 推荐来源类型

优先顺序如下：

1. 国家或行业协会公开指南
2. 医院官网公开发布的制度 / 培训文件
3. 公开设备说明书或操作手册
4. 公开教学课件或培训讲义

说明：

- demo 阶段优先选“公开 PDF / PPT / 网页导出 PDF”
- 尽量避免扫描质量很差的图片 PDF
- 优先选择可直接支撑 SOP、质控、设备操作、培训问答的资料

## 禁止来源

- 未授权内部制度原件
- 来源不明的扫描件
- 包含患者隐私的真实病例材料
- 无法说明版权或来源的网盘文件

## 文件命名规则

格式：

`[type]_[topic]_[version_or_year].[ext]`

示例：

- `guideline_pathology_qualitycontrol_2025.pdf`
- `sop_specimen_process_v1.pdf`
- `manual_stainer_operation_2024.pdf`
- `training_pathology_new_staff_2025.pptx`

推荐再补一个来源登记表，最少记录：

- 文件名
- 数据类型
- 来源机构
- 来源链接
- 发布时间或版本
- 去敏状态
- 对应演示场景

## 去敏规则

必须检查并移除：

- 患者姓名
- 检查号
- 住院号
- 联系方式
- 身份证号
- 可识别图像隐私信息

## 入库前检查项

- 来源可说明
- 文件命名符合规范
- 无重复页
- 无空白页
- 无明显 OCR 错误
- 无隐私字段
- 可归类到明确数据类型
- 可对应至少一个演示场景

## 推荐最小数据包结构

建议整理成以下结构：

```text
demo-data/
  guidelines/
  sops/
  manuals/
  training/
  sources.md
```

其中：

- `guidelines/` 放指南 / 规范
- `sops/` 放 SOP / 制度
- `manuals/` 放设备说明书
- `training/` 放培训资料
- `sources.md` 记录来源与用途

## 推荐最小数据集

- 2 到 3 份指南 / 规范
- 3 到 5 份 SOP / 流程文档
- 1 到 2 份设备说明书
- 1 份培训材料

## 推荐最小数据包清单

### A. 指南 / 规范

- 质控相关公开指南 1 份
- 标本处理或病理流程相关公开规范 1 份
- 可选补充：实验室质量管理相关规范 1 份

### B. SOP / 流程文档

- 标本接收流程 1 份
- 标本处理 / 染色流程 1 份
- 复核 / 质控流程 1 份
- 异常上报或设备异常处理流程 1 份
- 可选补充：培训签到 / 考核流程 1 份

### C. 设备说明书

- 某类染色设备或实验室设备公开手册 1 份
- 可选补充：常见报警处理说明 1 份

### D. 培训资料

- 新人入科培训资料 1 份

## 每类数据要支撑的场景

| 数据类型 | 至少支撑的场景 |
|---|---|
| 指南 / 规范 | 指南查询、规范引用回答、边界说明 |
| SOP / 流程文档 | SOP 查询、流程问答、培训问答 |
| 设备说明书 | 设备操作、异常处理 |
| 培训资料 | 培训问答、流程易错点总结 |

## 数据包验收口径

最小数据包完成时，至少满足：

1. 总文档数达到最小数据集要求
2. 每份文档都能说明来源
3. 每份文档命名符合规范
4. 每份文档都能映射到至少一个标准演示题
5. 无真实患者隐私字段

```md

|---|---|---|---|---|---|---|

```
