# 测试文档 (Test Documents)

本目录存放自动驾驶测试相关的场景库、测试用例和测试报告。

## 文档类型

### 1. 功能测试场景 (Functional Test Scenarios)
- **跟车场景**：ACC 自适应巡航、车距保持、速度跟随
- **变道场景**：车道变更、并线辅助、盲区监测
- **超车场景**：超车决策、超车执行、超车完成
- **转弯场景**：路口转弯、环岛通行、掉头场景
- **泊车场景**：平行泊车、垂直泊车、斜向泊车
- **交通灯场景**：红绿灯识别、停止线停车、绿灯起步
- **行人场景**：行人检测、行人避让、人行横道
- **交叉路口**：无信号路口、有信号路口、复杂路口

### 2. 安全测试场景 (Safety Test Scenarios)
- **紧急制动**：AEB 自动紧急制动、碰撞预警、制动距离
- **碰撞避免**：前向碰撞避免、侧向碰撞避免、后向碰撞避免
- **故障降级**：传感器故障、执行器故障、系统降级
- **接管测试**：人工接管、接管时间、接管成功率
- **极限工况**：极限加速、极限制动、极限转向
- **多车交互**：车车通信、协同避障、编队行驶

### 3. 边界测试场景 (Boundary Test Scenarios)
- **极端天气**：大雨、大雾、大雪、强光、夜间
- **复杂路况**：施工路段、临时路障、道路损坏
- **传感器遮挡**：摄像头遮挡、雷达遮挡、传感器污损
- **边界条件**：最小车距、最大速度、最小转弯半径
- **异常目标**：异形车辆、非机动车、动物、抛洒物
- **通信异常**：V2X 通信中断、GPS 信号丢失、网络延迟

### 4. 性能测试 (Performance Tests)
- **感知性能**：检测准确率、检测距离、检测延迟
- **规划性能**：规划时间、轨迹平滑度、舒适性
- **控制性能**：跟踪误差、响应时间、稳定性
- **系统性能**：CPU 占用率、内存占用、功耗

### 5. 仿真测试 (Simulation Tests)
- **场景仿真**：虚拟场景、交通流仿真、天气仿真
- **传感器仿真**：摄像头仿真、雷达仿真、激光雷达仿真
- **HIL 测试**：硬件在环测试、实时仿真、故障注入
- **SIL 测试**：软件在环测试、算法验证、参数优化

### 6. 实车测试 (Real Vehicle Tests)
- **封闭场地**：测试场地、标准场景、可重复测试
- **开放道路**：城市道路、高速公路、乡村道路
- **里程测试**：累计里程、场景覆盖、故障统计
- **用户测试**：用户体验、接受度、满意度

## 元数据标签

所有文档将被标记为：
- `document_type`: `test_doc`
- `test_type`: `functional` | `safety` | `boundary` | `performance` | `simulation` | `real_vehicle`
- `test_category`: 具体测试类别（如 `following`, `lane_change`, `emergency_brake`）

## 文档命名规范

建议格式：`test_{类型}_{场景名称}_{版本}.pdf`

示例：
- `test_functional_following_scenario_v1.0.pdf`
- `test_safety_emergency_brake_case_v2.1.pdf`
- `test_boundary_extreme_weather_report_v1.5.pdf`

## 测试文档内容要求

每个测试文档应包含：

### 测试场景定义
1. **场景描述**：场景名称、场景类型、场景目的
2. **初始条件**：车辆状态、环境条件、交通状况
3. **触发条件**：场景触发条件、关键事件
4. **预期行为**：系统预期响应、安全要求
5. **评价标准**：通过标准、失败标准、评分方法

### 测试用例
1. **用例编号**：唯一标识、版本号
2. **测试步骤**：详细步骤、操作说明
3. **输入数据**：测试输入、参数设置
4. **预期输出**：预期结果、性能指标
5. **实际结果**：测试结果、数据记录
6. **通过标准**：判定标准、容差范围

### 测试报告
1. **测试概述**：测试目的、测试范围、测试环境
2. **测试结果**：通过率、失败案例、性能数据
3. **问题分析**：问题描述、原因分析、影响评估
4. **改进建议**：优化方向、改进措施、后续计划
5. **附录**：测试数据、日志文件、截图视频
