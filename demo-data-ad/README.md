# 自动驾驶知识库 Demo 数据

本目录用于存放自动驾驶知识检索系统的演示数据包。

## 目录结构

- `sensors/`：传感器文档（摄像头、激光雷达、毫米波雷达、超声波雷达规格书和标定文档）
- `algorithms/`：算法文档（感知、规划、控制算法设计文档和技术报告）
- `regulations/`：法规文档（GB/T 国家标准、ISO 26262 功能安全标准、测试规范）
- `tests/`：测试文档（测试场景库、测试用例、测试报告）

## 文档要求

### 目标数量
- **总计**：至少 260 份文档
- **传感器文档**：>= 50 份
- **算法文档**：>= 80 份
- **法规文档**：>= 30 份
- **测试文档**：>= 100 份

### 文档类型

#### 1. 传感器文档 (sensors/)
- 摄像头规格书和标定文档
- 激光雷达（LiDAR）规格书和标定文档
- 毫米波雷达规格书和标定文档
- 超声波雷达规格书和标定文档
- 传感器融合方案文档

#### 2. 算法文档 (algorithms/)
- 感知算法：目标检测、车道线检测、障碍物识别、语义分割
- 规划算法：路径规划、轨迹规划、行为决策、运动规划
- 控制算法：横向控制、纵向控制、PID 控制、MPC 控制
- SLAM 算法文档

#### 3. 法规文档 (regulations/)
- GB/T 国家标准（自动驾驶分级、功能安全等）
- ISO 26262 功能安全标准
- 测试规范和认证要求
- 行业标准和企业标准

#### 4. 测试文档 (tests/)
- 功能测试场景（跟车、变道、超车等）
- 安全测试场景（紧急制动、碰撞避免、故障降级）
- 边界测试场景（极端天气、复杂路况、传感器遮挡）
- 测试用例和测试报告

## 元数据标签

每个文档在摄取时将被自动标记：
- `sensor_doc`：传感器文档
- `algorithm_doc`：算法文档
- `regulation_doc`：法规文档
- `test_doc`：测试文档

## 使用说明

1. 将文档按类型放入对应目录
2. 更新 `sources.md` 记录文档来源
3. 运行摄取脚本：
   ```bash
   python scripts/ingest.py \
     --collection ad_knowledge_v01 \
     --source demo-data-ad \
     --config config/settings.ad_knowledge.yaml
   ```

## 注意事项

- 所有文档应为公开资料或已获授权使用
- 禁止包含商业机密或敏感信息
- 文档格式支持：PDF、DOCX、TXT、MD
- 建议文档命名规范：`类型_名称_版本.pdf`
