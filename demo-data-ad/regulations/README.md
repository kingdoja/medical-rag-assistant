# 法规文档 (Regulation Documents)

本目录存放自动驾驶相关的法规标准和测试规范。

## 文档类型

### 1. 国家标准 (National Standards - GB/T)
- **自动驾驶分级**：GB/T 40429-2021 汽车驾驶自动化分级
- **功能安全**：GB/T 34590 系列 道路车辆功能安全
- **测试方法**：GB/T 相关测试标准
- **术语定义**：GB/T 术语和定义标准
- **性能要求**：GB/T 性能指标和要求

### 2. ISO 26262 功能安全标准
- **Part 1**：词汇
- **Part 2**：功能安全管理
- **Part 3**：概念阶段
- **Part 4**：产品开发：系统层面
- **Part 5**：产品开发：硬件层面
- **Part 6**：产品开发：软件层面
- **Part 7**：生产和运行
- **Part 8**：支持过程
- **Part 9**：以安全为导向的分析
- **Part 10**：指南
- **Part 11**：半导体应用指南
- **Part 12**：摩托车应用指南

### 3. 测试规范 (Test Specifications)
- **场景测试**：测试场景定义、场景库、场景覆盖
- **功能测试**：功能测试用例、测试方法、通过标准
- **安全测试**：安全测试规范、故障注入、降级测试
- **性能测试**：性能指标、测试环境、评价标准
- **仿真测试**：仿真平台、虚拟测试、HIL 测试
- **实车测试**：道路测试、封闭场地测试、开放道路测试

### 4. 行业标准和企业标准
- **SAE J3016**：自动驾驶分级标准（美国）
- **ISO/PAS 21448 (SOTIF)**：预期功能安全
- **ISO 21434**：网络安全工程
- **企业标准**：各大车企和科技公司的内部标准

### 5. 认证要求
- **型式认证**：车辆型式认证要求
- **准入管理**：智能网联汽车准入管理
- **测试认证**：第三方测试认证要求
- **数据安全**：数据安全管理要求

## 元数据标签

所有文档将被标记为：
- `document_type`: `regulation_doc`
- `regulation_type`: `national_standard` | `iso_standard` | `test_spec` | `industry_standard`
- `standard_number`: 标准编号（如 `GB/T 40429-2021`, `ISO 26262`）

## 文档命名规范

建议格式：`regulation_{类型}_{标准编号}_{名称}.pdf`

示例：
- `regulation_gb_40429-2021_driving_automation_classification.pdf`
- `regulation_iso_26262_part3_concept_phase.pdf`
- `regulation_test_functional_test_specification_v1.0.pdf`

## 文档内容要求

每个法规文档应包含：
1. **标准概述**：标准目的、适用范围、发布机构
2. **核心要求**：关键条款、技术要求、合规标准
3. **术语定义**：专业术语、缩写说明、概念解释
4. **实施指南**：实施方法、检查清单、常见问题
5. **参考案例**：应用案例、最佳实践、经验总结

## ASIL 等级说明

ISO 26262 定义的汽车安全完整性等级（ASIL）：
- **ASIL A**：最低安全等级
- **ASIL B**：低安全等级
- **ASIL C**：中等安全等级
- **ASIL D**：最高安全等级
- **QM**：质量管理（无 ASIL 要求）

## 文档权威性排序

在系统中，法规文档将按以下权威性排序：
1. **国家标准** (GB/T) - 最高权威性
2. **国际标准** (ISO, SAE) - 高权威性
3. **行业标准** - 中等权威性
4. **企业标准** - 参考权威性
