# 项目改造建议（求职版）

## 主推方向

将本项目包装为：

**病理 / 检验科医疗知识与质控助手**

## 为什么选这个方向

- 与你的医疗视觉检测落地经历闭环最强
- 与现有仓库的多模态摄取、混合检索、MCP 和评估能力最匹配
- 更适合投 AI 应用 / RAG / Agent / 医疗 AI 应用工程岗位
- 风险更低：强调知识、培训、质控与流程辅助，不碰自动诊断

## 一句话介绍

面向病理科 / 检验科内部知识管理、培训问答与流程质控场景，基于 RAG + MCP 实现指南、SOP、设备说明书和培训资料的可信检索、规范引用回答与 Agent 接入。

## 核心亮点

- 图文资料统一摄取与检索
- BM25 + Dense + RRF + rerank 混合召回
- 回答附引用，强调可信与可追溯
- 可通过 MCP 协议接入外部 Agent / Copilot
- 可观测与评估体系支撑持续迭代

## 简历写法参考

- 将通用模块化 RAG 底座垂直改造为病理 / 检验科医疗知识与质控助手，支撑指南、SOP、培训资料与设备说明书的统一检索和引用回答。
- 设计多源文档摄取链路，完成文档解析、语义切分、元数据增强、图片描述生成和增量入库。
- 实现 BM25 + Dense Retrieval + RRF 混合召回与重排，提升医疗术语、规范编号与自然语言问法混合场景下的检索准确性。
- 基于 MCP 协议封装知识检索与摘要能力，为外部 Agent / Copilot 提供可调用的知识工具。

## 讲述策略

推荐强调：

- 我不是做了一个通用聊天机器人
- 我是把通用 RAG 底座垂直化，变成了一个医疗内部工作流可用的知识助手
- 这个项目体现的是我把医疗场景理解和 LLM 工程能力结合起来做 AI 应用的能力
